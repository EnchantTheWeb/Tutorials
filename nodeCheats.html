ðŸ“š What Are Node.js and Express?

Node.js

Node.js is a runtime environment that lets you run JavaScript outside of a web browser. It's perfect for building server-side applications (the backend of websites). Think of it like a kitchen where you prepare food (your code runs here) before serving it to customers (users in their browsers).

Express

Express is a lightweight framework that makes building web servers with Node.js much easier. It helps you handle requests from users, route them to the right places, and send responses back. It's like a traffic controller for your web server!

ðŸŽ¯ Getting Started with Express

Installation & Setup

First, you need to create a project and install Express. Here's how:

Terminal Commands
npm init -y // Creates a package.json file (project configuration) npm install express // Downloads and installs Express
Note: npm stands for Node Package Manager. It's like an app store for JavaScript packages/libraries that you can install for your projects.

Your First Express Server

Here's a basic example of creating and starting a web server:

app.js
const express = require('express'); const app = express(); // Tell Express to listen on port 3000 const PORT = 3000; // Create a simple route app.get('/', (req, res) => { res.send('Hello, World!'); }); // Start the server app.listen(PORT, () => { console.log(`Server running on http://localhost:${PORT}`); });
Tip: Run this with node app.js in your terminal, then open your browser to http://localhost:3000 to see your server in action!

ðŸ”‘ Core Concepts

Routes

Routes are the paths users can visit on your website. When a user makes a request to a specific path, Express directs them to the appropriate code to handle that request.

Different HTTP Methods
// GET - Ask for data app.get('/users', (req, res) => { res.send('Here are all users'); }); // POST - Send/create new data app.post('/users', (req, res) => { res.send('User created!'); }); // PUT - Update existing data app.put('/users/1', (req, res) => { res.send('User updated!'); }); // DELETE - Remove data app.delete('/users/1', (req, res) => { res.send('User deleted!'); });
Note: These are called HTTP Methods or HTTP Verbs. They tell the server what action you want to perform.

Request (req) and Response (res)

Every route handler receives two objects: req (the incoming request from the user) and res (what you send back to the user).

Request and Response Example
app.get('/profile', (req, res) => { // req contains info about the user's request console.log(req.url); // The path they visited console.log(req.method); // GET, POST, etc. console.log(req.query); // URL parameters (?name=John) console.log(req.body); // Data sent in request body // Send a response back res.send('Profile page'); });

Middleware

Middleware is code that runs before your route handlers. Think of it like a security checkpoint that processes the request before sending it to your route. It's useful for authentication, logging, parsing data, and more.

Using Middleware
// Middleware that runs on every request app.use((req, res, next) => { console.log('A request was made to: ' + req.url); next(); // Pass control to the next middleware/route }); // Built-in middleware to parse JSON app.use(express.json()); app.post('/data', (req, res) => { res.json({message: 'Data received'}); });
Tip: The next() function tells Express "I'm done, move to the next middleware or route handler."

ðŸ“¤ Response Methods

These are different ways to send data back to the user:

Method What It Does Example
res.send() Sends text or HTML to the user res.send('Hello!')
res.json() Sends JSON data (perfect for APIs) res.json({name: 'John'})
res.status() Sets the HTTP status code res.status(404).send('Not found')
res.render() Renders a template file res.render('index', {data})
res.redirect() Redirects to another URL res.redirect('/home')
res.download() Sends a file to download res.download('file.pdf')

ðŸ”— URL Parameters and Query Strings

URL Parameters (in the path)

Parameters are variable parts of the URL that let you access specific items:

Example
// :id is a parameter - it can be any value app.get('/users/:id', (req, res) => { // Access the parameter value const userId = req.params.id; res.send(`Getting user ${userId}`); }); // URL: http://localhost:3000/users/42 // Output: "Getting user 42"

Query Strings (after the ?)

Query strings pass additional information in the URL after a question mark:

Example
// Access query parameters with req.query app.get('/search', (req, res) => { const searchTerm = req.query.q; const sortBy = req.query.sort; res.send(`Searching for: ${searchTerm}, sorted by: ${sortBy}`); }); // URL: http://localhost:3000/search?q=nodejs&sort=date // Output: "Searching for: nodejs, sorted by: date"

ðŸ’¡ Common Express Patterns

Setting Static Files

Serve static files like CSS, JavaScript, and images from a folder:

Serving Static Files
// Serve files from the 'public' folder app.use(express.static('public')); // Now these files are accessible: // http://localhost:3000/style.css (from public/style.css) // http://localhost:3000/script.js (from public/script.js)

Parsing Form Data

When users submit forms, you need to parse the data:

Parsing Data
// Middleware to parse JSON app.use(express.json()); // Middleware to parse form data app.use(express.urlencoded({extended: true})); app.post('/submit', (req, res) => { // Access form data console.log(req.body); res.send('Form received!'); });

Error Handling

Handle errors gracefully so your server doesn't crash:

Error Handling
// Route that doesn't exist app.use((req, res) => { res.status(404).send('Page not found'); }); // Error handling middleware (must be last) app.use((err, req, res, next) => { console.log(err); res.status(500).send('Something went wrong!'); });

ðŸŒŸ Tips for Beginners

1. Start Simple: Begin with basic routes and gradually add complexity. Don't try to build everything at once!
2. Use Console.log(): Print values to your terminal to debug. If you're not sure what's happening, log it out!
3. Test with Postman or Thunder Client: These tools let you test your routes without building a full frontend. Great for learning!
4. Understand the Request-Response Cycle: User sends request â†’ Express routes it â†’ Your code processes it â†’ You send response back. This is the foundation!
5. Read Error Messages: Node.js gives detailed error messages. Read them carefully - they usually tell you exactly what's wrong!
Common Beginner Mistake: Forgetting to call res.send(), res.json(), or next(). The user will be waiting forever if you don't send a response!

ðŸŽ“ Full Beginner Example

Here's a complete example that brings it all together:

Complete Basic Server (app.js)
const express = require('express'); const app = express(); // Middleware app.use(express.json()); app.use(express.static('public')); // Logging middleware app.use((req, res, next) => { console.log(`${new Date().toISOString()} - ${req.method} ${req.url}`); next(); }); // Routes app.get('/', (req, res) => { res.send('Welcome to my server!'); }); // Get all users app.get('/users', (req, res) => { res.json([ {id: 1, name: 'Alice'}, {id: 2, name: 'Bob'} ]); }); // Get specific user app.get('/users/:id', (req, res) => { const userId = req.params.id; res.json({id: userId, name: 'User ' + userId}); }); // Create new user app.post('/users', (req, res) => { const newUser = req.body; res.status(201).json({message: 'User created', user: newUser}); }); // Handle 404 errors app.use((req, res) => { res.status(404).send('Page not found'); }); // Start server const PORT = 3000; app.listen(PORT, () => { console.log(`Server running on http://localhost:${PORT}`); });
To run this: Save as app.js, then run node app.js in your terminal. Visit http://localhost:3000 in your browser!
