‚öôÔ∏èInitial Setup

git config --global user.name "Your Name"
Sets your name that will appear on your commits (saves to your code)
Note: Replace "Your Name" with your actual name in quotes
git config --global user.email "your.email@example.com"
Sets your email that will appear on your commits
Tip: Use the same email as your GitHub account
git config --list
Shows all your Git configuration settings

üÜïStarting a New Project

git init
Creates a new Git repository in your current folder
What it does: This tells Git to start tracking changes in this folder. Think of it as saying "Git, please watch this folder for me!"
git clone https://github.com/username/repository.git
Downloads a copy of an existing project from GitHub to your computer
Tip: Replace the URL with the actual repository URL you want to copy

üìùBasic Daily Commands

git status
Shows what files have changed and what's ready to be saved
Best Practice: Use this often! It's like checking your to-do list
git add filename.txt
Prepares a specific file to be saved (stages it)
Think of it as: Putting items in your shopping cart before checkout
git add .
Prepares ALL changed files to be saved at once
Warning: Make sure you want to add everything before using this!
git commit -m "Your message here"
Saves your changes with a description of what you did
Good commit messages: "Fix login bug" or "Add contact page" (be specific!)
git push
Uploads your saved changes to GitHub (or another remote server)
Think of it as: Backing up your work to the cloud
git pull
Downloads the latest changes from GitHub to your computer
Use this: Before starting work each day to get the latest updates

üåøBranching (Working on Different Versions)

git branch
Lists all branches (versions) of your project
What's a branch? It's like a parallel universe where you can make changes without affecting the main project
git branch new-feature
Creates a new branch called "new-feature"
git checkout new-feature
Switches to the "new-feature" branch
Tip: Now you're working in that branch, so changes won't affect the main code
git checkout -b another-feature
Creates AND switches to a new branch in one command
Shortcut: This combines creating and switching branches!
git merge new-feature
Combines the "new-feature" branch into your current branch
When to use: After you've tested your changes and want to add them to the main project
git branch -d new-feature
Deletes the "new-feature" branch after you're done with it
Warning: Only delete branches you don't need anymore!

üìúViewing History

git log
Shows a list of all previous saves (commits) with details
Tip: Press 'q' to exit this view
git log --oneline
Shows a simplified, one-line version of your commit history
Easier to read: Great for getting a quick overview
git diff
Shows exactly what changed in your files (line by line)

‚Ü©Ô∏èUndoing Changes

git restore filename.txt
Discards changes you made to a file (reverts to last saved version)
Warning: This permanently removes your unsaved changes!
git restore --staged filename.txt
Removes a file from the staging area (undoes 'git add')
What it does: Takes the file out of your "shopping cart" but keeps your changes
git reset HEAD~1
Undoes your last commit but keeps the changes in your files
Use this when: You committed too early and want to add more changes

‚òÅÔ∏èWorking with GitHub (Remote Repositories)

git remote add origin https://github.com/username/repo.git
Connects your local project to a GitHub repository
"origin" is just a nickname for the GitHub URL - it's the default name everyone uses
git remote -v
Shows which GitHub repositories your project is connected to
git push -u origin main
Uploads your "main" branch to GitHub for the first time
The -u flag: Sets up tracking so next time you can just type 'git push'
git fetch
Downloads changes from GitHub but doesn't apply them yet
Difference from pull: Fetch is like previewing updates, pull actually applies them

üîÑTypical Daily Workflow

Step 1: Check what's new on GitHub
git pull
Step 2: Make your changes to files
Work on your code in your text editor
Step 3: Check what you changed
git status
Step 4: Stage your changes
git add .
Step 5: Save your changes
git commit -m "Describe what you did"
Step 6: Upload to GitHub
git push
