What is React?

React is a JavaScript library for building user interfaces. Instead of writing plain HTML, you write components that can be reused and updated dynamically. Think of it like building with LEGO blocks - you create small pieces (components) that you can combine to build bigger things!

JSX Elements (HTML-like tags in React)

In React, you write JSX which looks like HTML but has some differences:

<div>

<div className="container"> <p>Hello World!</p> </div>

A container element, same as HTML div. Notice we use className instead of class!

<span>

<span style={{color: 'blue'}}> Inline text </span>

Inline text element. Styles use double curly braces and camelCase!

<h1> to <h6>

<h1>Main Title</h1> <h2>Subtitle</h2> <h3>Section Title</h3>

Heading elements for titles and subtitles, just like in HTML.

<p>

<p>This is a paragraph of text.</p>

Paragraph element for blocks of text.

<button>

<button onClick={handleClick}> Click me! </button>

Button element. Notice onClick (camelCase) instead of onclick!

<input>

<input type="text" value={inputValue} onChange={handleChange} />

Input field. In React, inputs are usually "controlled" with state.

<img>

<img src="/image.jpg" alt="Description" />

Image element. Always include alt text for accessibility!

<ul> & <li>

<ul> <li key="1">Item 1</li> <li key="2">Item 2</li> </ul>

Unordered list and list items. Each li needs a unique key prop!

React Components

Components are the building blocks of React apps. They're like custom HTML tags that you create!

Function Component (Most Common)

function MyComponent() { return ( <div> <h1>Hello from my component!</h1> </div> ); } // Using the component <MyComponent />

Component with Props (Properties)

function Greeting(props) { return <h1>Hello, {props.name}!</h1>; } // Using it <Greeting name="Alice" />
Props are like function parameters - they let you pass data into components!

React Hooks (Special Functions)

useState - Managing State

import { useState } from 'react'; function Counter() { const [count, setCount] = useState(0); return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count + 1)}> Increase </button> </div> ); }
useState lets you add state (changeable data) to your component. It returns the current value and a function to update it.

useEffect - Side Effects

import { useEffect, useState } from 'react'; function DataLoader() { const [data, setData] = useState(null); useEffect(() => { // This runs when component loads console.log('Component mounted!'); // Cleanup function (optional) return () => { console.log('Component unmounting!'); }; }, []); // Empty array = run once return <div>{data}</div>; }
useEffect lets you perform side effects like API calls, subscriptions, or manually changing the DOM.

Event Handling

onClick

<button onClick={handleClick}> Click me </button>

Handles button clicks

onChange

<input onChange={handleChange} value={text} />

Handles input changes

onSubmit

<form onSubmit={handleSubmit}> <input type="submit" /> </form>

Handles form submissions

onMouseOver

<div onMouseOver={handleHover}> Hover me! </div>

Handles mouse hover events

Conditional Rendering

Using && Operator

function WelcomeMessage({ isLoggedIn }) { return ( <div> {isLoggedIn && <h1>Welcome back!</h1>} {!isLoggedIn && <h1>Please log in</h1>} </div> ); }

Using Ternary Operator

function StatusMessage({ isOnline }) { return ( <div> {isOnline ? ( <span style={{color: 'green'}}>Online</span> ) : ( <span style={{color: 'red'}}>Offline</span> )} </div> ); }

Lists and Keys

function TodoList({ todos }) { return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} </li> ))} </ul> ); } // Using it const todos = [ { id: 1, text: 'Learn React' }, { id: 2, text: 'Build an app' } ]; <TodoList todos={todos} />
Always provide a unique key prop when rendering lists. This helps React track changes efficiently!

Common Patterns

Handling Form Input

function ContactForm() { const [name, setName] = useState(''); const [email, setEmail] = useState(''); const handleSubmit = (e) => { e.preventDefault(); console.log('Name:', name, 'Email:', email); }; return ( <form onSubmit={handleSubmit}> <input type="text" placeholder="Name" value={name} onChange={(e) => setName(e.target.value)} /> <input type="email" placeholder="Email" value={email} onChange={(e) => setEmail(e.target.value)} /> <button type="submit">Submit</button> </form> ); }

Loading State

function DataComponent() { const [data, setData] = useState(null); const [loading, setLoading] = useState(true); useEffect(() => { // Simulate API call setTimeout(() => { setData('Hello from API!'); setLoading(false); }, 2000); }, []); return ( <div> {loading ? ( <p>Loading...</p> ) : ( <p>{data}</p> )} </div> ); }

Key Differences from HTML

Attributes

  • class ‚Üí className
  • for ‚Üí htmlFor
  • onclick ‚Üí onClick
  • All event handlers use camelCase

Styles

// HTML style="color: red; font-size: 16px;" // React style={{color: 'red', fontSize: '16px'}}

Styles are objects with camelCase properties

Self-Closing Tags

// HTML <input type="text"> <br> // React <input type="text" /> <br />

All tags must be closed in JSX

JavaScript Expressions

const name = 'World'; const element = <h1>Hello, {name}!</h1>;

Use curly braces {} to embed JavaScript

Complete Example App

import { useState } from 'react'; function App() { const [todos, setTodos] = useState([]); const [inputValue, setInputValue] = useState(''); const addTodo = () => { if (inputValue.trim()) { setTodos([...todos, { id: Date.now(), text: inputValue, completed: false }]); setInputValue(''); } }; const toggleTodo = (id) => { setTodos(todos.map(todo => todo.id === id ? {...todo, completed: !todo.completed} : todo )); }; return ( <div className="app"> <h1>My Todo App</h1> <div> <input type="text" value={inputValue} onChange={(e) => setInputValue(e.target.value)} placeholder="Add a todo..." /> <button onClick={addTodo}>Add</button> </div> <ul> {todos.map(todo => ( <li key={todo.id} onClick={() => toggleTodo(todo.id)} style={{ textDecoration: todo.completed ? 'line-through' : 'none', cursor: 'pointer' }} > {todo.text} </li> ))} </ul> </div> ); } export default App;
This example shows a complete Todo app using useState, event handling, conditional rendering, and lists!

Quick Tips for Beginners

Start with function components - they're simpler and more modern than class components.
Always use useState for data that can change, and pass static data as props.
Remember that React re-renders components when state changes - this is how your UI updates!
Use the React Developer Tools browser extension to debug your components.
Components must return exactly one parent element. Use a <div> or React Fragment <></> to wrap multiple elements.
