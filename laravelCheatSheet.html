üìö What is Laravel?

Laravel is a PHP framework that makes building websites easier. Think of it as a toolkit with pre-built tools and patterns that help you create web applications faster and more securely.

üéØ Routing (How URLs Work)

Routes tell Laravel what to do when someone visits a specific URL on your website.

Basic Routes

Route::get('/home', function () { return 'Welcome Home!'; });
What this does: When someone visits yoursite.com/home, they see "Welcome Home!"

Different HTTP Methods

Route::get('/page', ...); // View a page Route::post('/submit', ...); // Submit a form Route::put('/update', ...); // Update data Route::delete('/remove', ...); // Delete data

Route with Parameters

Route::get('/user/{id}', function ($id) { return 'User ID: ' . $id; });
What this does: The {id} is a placeholder. If someone visits /user/5, $id will equal 5.

Named Routes (For Easy Reference)

Route::get('/profile', ...)->name('profile');
Tip: Use named routes so if you change the URL later, you don't have to update it everywhere!

üéÆ Controllers (Organizing Your Code)

Controllers handle the logic for your routes. Instead of putting code in routes, you put it in controller methods.

Creating a Controller

php artisan make:controller UserController
What this does: Creates a new controller file where you can organize your code.

Basic Controller Example

class UserController extends Controller { public function index() { return 'List of users'; } public function show($id) { return 'User: ' . $id; } }

Using Controller in Routes

Route::get('/users', [UserController::class, 'index']); Route::get('/users/{id}', [UserController::class, 'show']);

üé® Blade Templates (Creating Views)

Blade is Laravel's templating engine. It lets you write HTML with special tags for dynamic content.

Displaying Variables

{{ $name }} // Displays the variable and escapes HTML (safe) {!! $html !!} // Displays raw HTML (use carefully!)

If Statements

@if($user) <p>Welcome, {{ $user->name }}!</p> @elseif($guest) <p>Welcome, Guest!</p> @else <p>Please log in.</p> @endif

Loops

@foreach($users as $user) <p>{{ $user->name }}</p> @endforeach @for($i = 0; $i < 10; $i++) <p>Number: {{ $i }}</p> @endfor

Layout Inheritance

// layout.blade.php (master template) <html> <body> @yield('content') </body> </html> // page.blade.php (child template) @extends('layout') @section('content') <h1>My Page Content</h1> @endsection
What this does: Creates a master template you can reuse, so you don't repeat the same HTML on every page!

üíæ Database (Models & Eloquent)

Models represent your database tables. Eloquent is Laravel's way of working with databases using simple PHP code instead of SQL.

Creating a Model

php artisan make:model User -m // -m also creates a migration

Basic Model Usage

// Get all records $users = User::all(); // Find by ID $user = User::find(1); // Create new record $user = new User(); $user->name = 'John'; $user->email = 'john@example.com'; $user->save(); // Update record $user = User::find(1); $user->name = 'Jane'; $user->save(); // Delete record $user = User::find(1); $user->delete();

Querying Data

// Where clause $users = User::where('active', 1)->get(); // Multiple conditions $users = User::where('active', 1) ->where('age', '>', 18) ->get(); // Order and limit $users = User::orderBy('name') ->take(10) ->get();

üìù Migrations (Database Structure)

Migrations are like version control for your database. They let you create and modify database tables.

Creating a Migration

php artisan make:migration create_posts_table

Migration Example

public function up() { Schema::create('posts', function (Blueprint $table) { $table->id(); $table->string('title'); $table->text('content'); $table->timestamps(); // Adds created_at & updated_at }); }

Running Migrations

php artisan migrate // Run migrations php artisan migrate:rollback // Undo last migration php artisan migrate:fresh // Drop all tables and re-run

‚úÖ Validation (Checking User Input)

Always validate data before saving it to prevent errors and security issues.

$validated = $request->validate([ 'name' => 'required|max:255', 'email' => 'required|email|unique:users', 'age' => 'required|integer|min:18', 'password' => 'required|min:8|confirmed', ]);

Common Validation Rules

  • required - Field must have a value
  • email - Must be a valid email format
  • unique:table - Value must be unique in that table
  • min:5 - Minimum length or value
  • max:100 - Maximum length or value
  • confirmed - Must match a field_confirmation field
  • integer - Must be a whole number

üîó Relationships (Connecting Tables)

Define how your database tables relate to each other.

One to Many (e.g., User has many Posts)

// In User model public function posts() { return $this->hasMany(Post::class); } // In Post model public function user() { return $this->belongsTo(User::class); } // Using it $user = User::find(1); $posts = $user->posts; // Get all user's posts

üéì Quick Start Workflow

Typical Laravel Development Flow:
  1. Create a migration for your database table
  2. Run the migration to create the table
  3. Create a model for that table
  4. Create a controller to handle the logic
  5. Add routes to connect URLs to controller methods
  6. Create Blade views to display your pages
  7. Test everything and celebrate! üéâ
