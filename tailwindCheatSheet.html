üî≤ Spacing (Margin & Padding)

Margin (Space Outside)

Margin creates space outside an element - between it and other elements.

m-0, m-1, m-2, m-4, m-6, m-8, m-10, m-12, m-14, m-16, m-20, m-24, m-28, m-32 mt-4 mb-8 ml-2 mr-6 // top, bottom, left, right mx-auto my-4 // horizontal, vertical mx-2 my-6 // x-axis (left+right), y-axis (top+bottom)
Quick Tip: m-4 = margin on all sides | mt-4 = margin top only | mx-auto = centers element horizontally
Remember: Each unit = 0.25rem (4 pixels) ‚Üí m-4 = 1rem = 16px

Padding (Space Inside)

Padding creates space inside an element - between the content and the border.

p-0, p-1, p-2, p-4, p-6, p-8, p-10, p-12, p-14, p-16, p-20, p-24, p-28, p-32 pt-4 pb-8 pl-2 pr-6 // top, bottom, left, right px-6 py-4 // x-axis (left+right), y-axis (top+bottom)
Quick Tip: p-4 = padding all sides | pb-8 = padding bottom | px-6 = left and right padding

Gap (Space Between Items)

Gap creates even spacing between flex or grid items.

gap-0, gap-1, gap-2, gap-3, gap-4, gap-6, gap-8, gap-10, gap-12, gap-14, gap-16, gap-20, gap-24 gap-x-4 gap-y-8 // horizontal gap, vertical gap

üìè Sizing (Width & Height)

Width

w-auto, w-full // auto size, 100% width w-screen // full viewport width w-1/2, w-1/3, w-1/4, w-2/3 // fractions (50%, 33%, 25%, 66%) w-12, w-16, w-20, w-24, w-32, w-40, w-48, w-56, w-64, w-72, w-80, w-96 max-w-sm, max-w-md, max-w-lg, max-w-xl, max-w-2xl, max-w-3xl, max-w-4xl max-w-full, max-w-prose
Pro Tip: max-w-4xl is perfect for readability - it limits text width on large screens!

Height

h-auto, h-full // auto size, 100% of parent h-screen // 100% of viewport height h-12, h-16, h-20, h-24, h-32, h-40, h-48, h-56, h-64, h-80, h-96 min-h-screen // minimum height of viewport min-h-full // minimum 100% height max-h-64, max-h-96 // maximum heights

Aspect Ratio

aspect-auto, aspect-square, aspect-video aspect-[1/1], aspect-[4/3], aspect-[16/9], aspect-[3/2]
Use Case: Great for images and videos that need to maintain proportion!

üéØ Display & Layout

Display Types

block // takes full width, new line inline // flows with text inline-block // inline but accepts width/height hidden // display: none (hidden) flex // flexbox layout grid // grid layout

Position

static // default - normal flow relative // positioned relative to normal spot absolute // positioned relative to parent fixed // positioned relative to viewport (stays when scrolling) sticky // stays when scrolling past it top-0, bottom-0, left-0, right-0 // positioning offsets inset-0 // all sides = 0 (fills parent)

Overflow & Visibility

overflow-visible // default - content shows outside overflow-hidden // cuts off content that doesn't fit overflow-auto // scrollbar only if needed overflow-x-auto, overflow-y-auto // scroll on specific axis visible, invisible, hidden // visibility options

Z-index (Layering)

z-0, z-10, z-20, z-30, z-40, z-50 z-auto // auto z-index
Note: Higher number = appears on top. Use for modals, dropdowns, floating elements!

üñºÔ∏è Borders & Shadows

Border

border // 1px border all sides border-2 // 2px border border-4, border-8 // thicker borders border-t, border-b, border-l, border-r // specific sides border-gray-300 // color of border border-blue-500 // blue border border-slate-200 // light gray border

Rounded Corners

Pro Tip: rounded-lg is the modern standard for most elements!

üì¶ Flexbox

Flex Container

flex // enables flexbox layout flex-row // items left to right (default) flex-col // items top to bottom flex-row-reverse // right to left flex-col-reverse // bottom to top flex-wrap // items wrap to next line if needed flex-nowrap // items stay on one line

Justify Content (Horizontal Alignment)

justify-start // items at start (left) justify-center // items centered justify-end // items at end (right) justify-between // space between items justify-around // space around items justify-evenly // equal space everywhere

Align Items (Vertical Alignment)

items-start // items at top items-center // items centered vertically items-end // items at bottom items-baseline // align text baselines items-stretch // items stretch to fill height
Magic Combo: flex justify-center items-center = PERFECT CENTERING both ways!

Flex Items

flex-1 // takes equal space (MOST USEFUL!) flex-auto // takes space (different than flex-1) flex-initial // doesn't grow, shrinks if needed flex-none // doesn't grow or shrink

‚öôÔ∏è Grid

Grid Container

grid // enables grid layout grid-cols-1 // 1 column grid-cols-2 // 2 columns grid-cols-3 // 3 columns grid-cols-4, grid-cols-5, grid-cols-6, grid-cols-12 grid-rows-1, grid-rows-3, grid-rows-6 // rows

Responsive Grid (MOBILE-FIRST!)

grid-cols-1 // 1 column on mobile md:grid-cols-2 // 2 columns on medium screens lg:grid-cols-3 // 3 columns on large screens xl:grid-cols-4 // 4 columns on extra large screens // Complete responsive example: grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6
Best Practice: Always design mobile-first (start with 1 column), then expand for larger screens!

Grid Items (Spanning)

col-span-1 // spans 1 column col-span-2 // spans 2 columns col-span-3, col-span-4, col-span-12 // spans multiple columns row-span-2, row-span-3 // spans multiple rows

‚ú® Effects & Transforms

Transform Scale (Zoom)

scale-50 // 50% size scale-75 // 75% size scale-100 // normal size scale-110 // 110% size scale-125, scale-150 // larger hover:scale-105 // slightly bigger on hover

Transform Rotate

rotate-0 // no rotation rotate-45 // 45 degree rotation rotate-90, rotate-180 // 90 degrees, 180 degrees -rotate-45 // -45 degrees (counter-clockwise)

Transition (Animation Smoothness)

transition // smooth transition on hover transition-all // transitions all properties transition-colors // only color transitions smooth transition-transform // only transform transitions smooth duration-150, duration-300, duration-500, duration-1000 // milliseconds ease-linear // constant speed ease-in, ease-out // start/end slow ease-in-out // slow start and end
Pro Combo: transition-all duration-300 hover:scale-110 = smooth professional hover effect!

Built-in Animations

animate-spin // rotation animation animate-ping // grows and fades (like radar ping) animate-pulse // fades in and out animate-bounce // bounces up and down animate-none // no animation

üì± Responsive Design

Breakpoints (Screen Sizes)

sm 640px // small phones and tablets md 768px // larger tablets lg 1024px // small laptops xl 1280px // normal desktops 2xl 1536px // large monitors // Backwards: not at this breakpoint and down max-sm // smaller than sm max-md // smaller than md

Common Responsive Patterns

// Responsive grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 // Responsive text size text-sm md:text-base lg:text-lg // Responsive padding p-4 md:p-6 lg:p-8 // Hide/show at breakpoints hidden md:block // hidden until medium screen block md:hidden // hidden from medium up // Responsive flex direction flex-col md:flex-row // column on mobile, row on desktop
Key Rule: Use prefixes (sm:, md:, lg:, xl:, 2xl:) on ANY class to make it responsive!

üéØ Hover & Interactive States

Hover States

Focus States (For Inputs)

focus:outline-none // removes outline focus:ring-2 // adds colored ring focus:ring-blue-500 // ring color focus:bg-white // changes background when focused // Example on input: focus:border-blue-500 focus:ring-2 focus:ring-blue-200

Other States

active:scale-95 // when being clicked disabled:opacity-50 // when disabled group-hover:text-white // child reacts to parent hover first:border-t-0 // first item in list last:border-b-0 // last item in list

üåü Tips for Beginners

1. Start with Common Classes
Focus on: p-, m-, w-, h-, bg-, text-, rounded-, shadow-, flex, grid, gap. These handle 80% of styling!
2. Mobile-First Approach
Always design mobile first (no prefix), then add breakpoints (sm:, md:, lg:) for larger screens.
3. Use Browser DevTools
Inspect elements to see which classes are applied. This helps you learn and debug!
4. Check Official Docs
Tailwind CSS docs are amazing! Visit tailwindcss.com for complete reference.
Common Mistake: Not using responsive prefixes. Always test your design on mobile, tablet, and desktop!
