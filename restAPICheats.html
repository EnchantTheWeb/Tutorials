Your Beginner's Guide to Understanding REST APIs

üìö What is a REST API?

REST stands for Representational State Transfer. Think of an API (Application Programming Interface) as a waiter in a restaurant:

  • You (the client) tell the waiter what you want
  • The waiter takes your request to the kitchen (the server)
  • The kitchen prepares your order
  • The waiter brings back your food (the response)
üí° Beginner Tip: REST APIs let different applications talk to each other over the internet using standard HTTP methods, just like web browsers do!

üîß HTTP Methods (Verbs)

These are the main actions you can perform with a REST API:

GET Retrieve Data

What it does: Asks the server to send you information. Like asking "Can I see the menu?"

Use when: You want to read/view data without changing anything

GET /api/users/123 // Gets information about user with ID 123

Note: GET requests should NEVER modify data on the server

POST Create New Data

What it does: Sends new information to the server to create something. Like submitting a new order.

Use when: You want to add new data (create a new user, post a comment, etc.)

POST /api/users Body: { "name": "John", "email": "john@example.com" } // Creates a new user
PUT Update/Replace Data

What it does: Replaces existing information completely. Like rewriting an entire document.

Use when: You want to update ALL fields of an existing resource

PUT /api/users/123 Body: { "name": "John Smith", "email": "john@example.com", "age": 30 } // Replaces ALL user data for user 123

Important: Usually requires sending ALL fields, even unchanged ones

PATCH Partial Update

What it does: Updates only specific parts of existing data. Like editing one paragraph in a document.

Use when: You only want to change specific fields

PATCH /api/users/123 Body: { "email": "newemail@example.com" } // Only updates the email, leaves other fields unchanged
DELETE Remove Data

What it does: Removes data from the server. Like canceling an order.

Use when: You want to delete a resource

DELETE /api/users/123 // Deletes user with ID 123

Warning: This action is usually permanent!

üîë Common API Components

Base URL / Endpoint

The main address of the API

https://api.example.com/v1
Resource Path

The specific data you want to access

/users/123/posts // Gets posts from user 123
Query Parameters

Extra filters or options (after the ?)

/users?age=25&city=Boston // Finds users who are 25 and live in Boston
Headers

Extra information about your request

Content-Type: application/json Authorization: Bearer your-token-here
Request Body

Data you're sending (used with POST, PUT, PATCH)

{ "username": "johndoe", "email": "john@example.com" }

‚ú® REST API Best Practices

1. Use Nouns for Resources: /users not /getUsers
2. Use Plural Names: /users not /user
3. Use HTTP Methods Correctly: GET for reading, POST for creating, etc.
4. Return Appropriate Status Codes: 200 for success, 404 for not found, etc.
5. Version Your API: /v1/users or /v2/users
